Panduan: Cara Install Terminal Git Bash (Windows)

Langkah demi langkah cepat untuk memasang Git Bash dan mulai menggunakan Git di Windows.

Ringkasan singkat

Git for Windows menyediakan Git Bash — terminal yang memberikan lingkungan mirip UNIX di Windows. Panduan ini untuk pengguna Windows (Win10/Win11).

1. Unduh installer

  1. Buka situs resmi gitforwindows.org.
  2. Klik tombol Download untuk Windows. Anda akan mendapatkan file bernama seperti Git-*-64-bit.exe (atau 32-bit jika menggunakan arsitektur lama).
  3. Simpan file di folder Downloads lalu jalankan dengan meng-klik dua kali.

2. Jalankan installer — pilihan penting

Ikuti wizard installer. Pada beberapa layar akan muncul opsi; berikut rekomendasi yang aman untuk pemula:

  1. Select Components: biarkan opsi default (Git Bash, Git GUI). Centang "Windows Explorer integration" jika ingin tombol klik kanan "Git Bash Here".
  2. Choosing the default editor used by Git: pilih editor yang Anda sukai (default seringnya Vim). Jika tidak familiar, pilih Use Visual Studio Code as Git's default editor jika menggunakan VS Code.
  3. Adjusting your PATH environment:
    • Rekomendasi: Git from the command line and also from 3rd-party software — ini menambahkan git ke PATH sehingga dapat dipakai di Command Prompt, PowerShell, serta program lain.
  4. Choosing HTTPS transport backend: gunakan default Use the OpenSSL library.
  5. Configuring line ending conversions:
    • Rekomendasi umum untuk proyek lintas platform: Checkout as-is, commit Unix-style line endings (atau opsi "Checkout Windows-style, commit Unix-style" jika proyek khusus Windows).
  6. Lanjutkan sampai selesai dan klik Finish.

3. Menjalankan Git Bash

  1. Buka menu Start → ketik Git Bash → Enter.
  2. Atau di File Explorer: klik kanan → Git Bash Here (jika integrasi Explorer diaktifkan saat instalasi).

Git Bash akan membuka jendela terminal dengan prompt mirip UNIX (bash). Anda dapat menjalankan perintah Git dan perintah shell dasar.

4. Verifikasi dan konfigurasi awal

Buka Git Bash lalu jalankan perintah berikut untuk memastikan Git terpasang dan mengatur identitas Anda:

git --version
# contoh output: git version 2.###.##
git config --global user.name "Nama Anda"
git config --global user.email "email@domain.com"

Untuk mengecek konfigurasi:

git config --list

5. Opsi lanjutan yang sering dipakai

  • Ganti editor default Git: 
    git config --global core.editor "code --wait"
  • Menghasilkan SSH key (untuk GitHub / GitLab): 
    ssh-keygen -t ed25519 -C "email@domain.com"
# lalu tambahkan isi ~/.ssh/id_ed25519.pub ke akun Git hosting Anda
  • Cache credential (agar tidak memasukkan password berulang): 
    git config --global credential.helper manager-core

6. Troubleshooting singkat

  • Perintah git tidak ditemukan: pastikan Anda memilih opsi menambahkan ke PATH saat instalasi atau ulangi instalasi dan pilih opsi tersebut.
  • Masalah baris akhir (CRLF/LF): gunakan salah satu opsi line ending saat instalasi sesuai kebutuhan tim/project.
  • SSH key tidak berfungsi: pastikan agen ssh berjalan, dan kunci publik sudah ditambahkan ke layanan Git (GitHub/GitLab).

7. Tes cepat — buat repo lokal

mkdir demo-git
cd demo-git
git init
echo "Halo" > README.md
git add README.md
git commit -m "Initial commit"
# cek status
git status

Jika langkah di atas berhasil tanpa error, Git Bash Anda siap untuk bekerja dengan repositori.
